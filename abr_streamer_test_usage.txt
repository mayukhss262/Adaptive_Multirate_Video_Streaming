## Key Design Decisions

1. **Deterministic Tests**: All tests use `FakeNetworkSimulator` and `FakeABRController` with controlled inputs for reproducibility

2. **No Randomness**: Tests avoid randomness; network simulator can use fixed seeds for deterministic behavior

3. **Logging**: Tests use Python `logging` for detailed diagnostics

4. **Detailed Assertions**: All assertions include context (expected vs actual values with relevant variables)

5. **Exit Codes**: 
   - Exit 0 = All tests passed
   - Exit 1 = One or more tests failed

6. **Fixtures**: `conftest.py` provides reusable fixtures:
   - `default_bitrate_ladder`: [400, 1200, 3000, 6000] kbps
   - `segment_duration`: 2.0 seconds (from config.py)
   - `fake_network_constant()`: Creates constant bandwidth network
   - `fake_network_sequence()`: Creates predefined bandwidth sequence
   - `fake_abr()`: Creates fake ABR returning fixed bitrate
   - `deterministic_network()`: Creates real NetworkSimulator with fixed seed

The test suite rigorously validates `abr_algorithm.py` and `streamer.py` with deterministic correctness validation.
